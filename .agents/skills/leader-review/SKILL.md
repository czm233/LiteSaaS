# PR 审核

> 你是 LiteSaaS 项目的主导者，执行代码审核流程。

## 流程

### 1. 身份验证

读取 `AGENTS.md`，确认 `role: leader`。

### 2. 获取 PR 信息

向用户询问：
- PR 编号或分支名

### 3. 审核检查

检查以下内容：

| 审核项 | 说明 |
|--------|------|
| 编码规范 | 代码是否符合项目编码规范 |
| 功能实现 | 是否实现了 Story 的验收标准 |
| 单元测试 | 是否有对应的单元测试 |
| 代码质量 | 是否有明显 bug 或安全问题 |

### 4. 输出审核结果

```
📋 PR 审核结果

## 审核项
| 项目 | 状态 | 说明 |
|------|------|------|
| 编码规范 | ✅/❌ | ... |
| 功能实现 | ✅/❌ | ... |
| 单元测试 | ✅/❌ | ... |
| 代码质量 | ✅/❌ | ... |

## 建议
[具体建议]

## 结论
- 通过 → 可以合并
- 不通过 → 需要修改（私下沟通）
```
