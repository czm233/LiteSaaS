## ADR-001: 技术栈选择

**状态**: ✅ 已确定

**日期**: 2026-04-01

**决策者**: czm

### 背景

项目是一个企业管理系统 SaaS 平台，需要支持多租户、高可用、易扩展。

### 决策

| 层面 | 选择 | 理由 |
|------|------|------|
| 后端运行时 | Node.js 20+ | 团队熟悉，生态丰富 |
| 后端框架 | Koa.js | 轻量、灵活、中间件机制 |
| ORM | Prisma | 类型安全、迁移管理方便 |
| 数据库 | MySQL 8.0+ | 成熟稳定、社区支持好 |
| 前端框架 | React 18+ / Vue 3+ | 待确定 |
| 构建工具 | Vite | 快速、现代 |
| 包管理器 | pnpm | 节省磁盘空间、依赖管理好 |

### 待确定

- [ ] 前端框架：React 还是 Vue？
- [ ] UI 组件库：Ant Design 还是 Element Plus？
- [ ] 状态管理方案
- [ ] 部署方案

## ADR-002: Git 工作流

**状态**: ✅ 已确定

**日期**: 2026-04-01

**决策者**: czm

### 决策

采用 **功能分支工作流**：

- `main` 分支受保护，禁止直接推送
- 协作者在自己的功能分支开发
- 通过 Pull Request 合并代码
- 主导者负责代码审核和合并

详见 [Git 工作流指南](./GIT-WORKFLOW.md)
