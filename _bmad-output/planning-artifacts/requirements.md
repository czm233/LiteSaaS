## 1. 产品定位

### 1.1 产品概述

LiteSaaS 是一个**可复用的多租户后台管理系统**，提供用户管理、权限管理、组织架构管理等核心能力，可作为各类 SaaS 产品的基础框架。

### 1.2 目标用户

- **SaaS 运营方**：使用本系统管理所有租户和用户
- **企业客户（租户）**：使用本系统管理自己的组织、用户和权限

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 快速启动 | 复制即可使用，无需从零搭建 |
| 权限灵活 | 5级权限 + 3种粒度，满足复杂场景 |
| 多租户隔离 | 数据安全隔离，互不干扰 |
| 可扩展 | 模块化设计，易于扩展新功能 |

## 2. 核心功能

### 2.1 功能清单

| 模块 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| **认证** | 登录/登出 | P0 | 支持选择租户、用户名密码登录 |
| **租户管理** | 租户 CRUD | P0 | 仅系统管理员可操作 |
| **组织管理** | 组织 CRUD | P0 | 支持多级组织（默认3级，可扩展） |
| **用户管理** | 用户 CRUD | P0 | 用户基本信息、状态管理 |
| **角色管理** | 角色 CRUD | P0 | 角色定义、权限分配 |
| **权限管理** | 权限配置 | P0 | 菜单级、按钮级、数据级权限 |
| **字典管理** | 字典 CRUD | P1 | 通用键值对配置 |
| **个人中心** | 修改密码 | P1 | 用户修改自己的密码 |

### 2.2 功能详情

#### 2.2.1 认证模块

| 功能 | 描述 |
|------|------|
| 登录 | 选择租户 → 输入用户名密码 → JWT 认证 |
| 登出 | 清除 Token，返回登录页 |
| 获取用户信息 | 返回当前用户的基本信息和权限 |

#### 2.2.2 租户管理

| 功能 | 描述 |
|------|------|
| 创建租户 | 输入租户名称、标识，创建新租户 |
| 编辑租户 | 修改租户基本信息 |
| 禁用/启用租户 | 控制租户的可用状态 |
| 删除租户 | 删除租户及其所有数据（谨慎操作） |

#### 2.2.3 组织管理

| 功能 | 描述 |
|------|------|
| 创建组织 | 在某个组织下创建子组织 |
| 编辑组织 | 修改组织名称、描述 |
| 删除组织 | 删除组织（需先处理下级和关联用户） |
| 组织树 | 以树形结构展示组织层级 |

#### 2.2.4 用户管理

| 功能 | 描述 |
|------|------|
| 创建用户 | 创建用户并分配到组织、角色 |
| 编辑用户 | 修改用户基本信息 |
| 禁用/启用用户 | 控制用户的登录状态 |
| 重置密码 | 重置用户密码 |
| 删除用户 | 删除用户（谨慎操作） |

#### 2.2.5 角色管理

| 功能 | 描述 |
|------|------|
| 创建角色 | 定义角色名称、描述 |
| 分配权限 | 为角色分配菜单、按钮、数据权限 |
| 编辑角色 | 修改角色信息 |
| 删除角色 | 删除角色（需先处理关联用户） |

#### 2.2.6 权限管理

| 权限类型 | 描述 |
|----------|------|
| **菜单权限** | 控制用户能看到哪些菜单 |
| **按钮权限** | 控制用户能看到/操作哪些按钮 |
| **数据权限** | 控制用户能看到哪些数据 |

## 3. 权限模型

### 3.1 权限层级（5级）

```
┌─────────────────────────────────────────────────────────────┐
│  1级 - 系统管理员（SaaS 运营方）                             │
│  ├── ✅ 管理所有账户（创建、禁用、删除）                      │
│  ├── ✅ 查看所有配置                                         │
│  ├── ✅ 控制租户的启用/禁用                                  │
│  └── ❌ 不能修改租户的业务数据（防止滥用）                    │
├─────────────────────────────────────────────────────────────┤
│  2级 - 租户管理员（企业客户）                                │
│  ├── ✅ 管理本租户内的所有账户                               │
│  ├── ✅ 配置本租户的业务（组织、角色、权限等）                │
│  └── ⚠️ 受1级控制，可被禁用                                  │
├─────────────────────────────────────────────────────────────┤
│  3级 - 组织管理员                                           │
│  ├── ✅ 管理本组织及下级组织的账户                           │
│  └── ⚠️ 受2级控制                                           │
├─────────────────────────────────────────────────────────────┤
│  4级 - 部门管理员                                           │
│  └── 管理本部门的账户和配置                                  │
├─────────────────────────────────────────────────────────────┤
│  5级 - 普通用户                                             │
│  └── 只能访问被授权的功能和数据                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 核心规则

| 规则 | 说明 |
|------|------|
| **账户管理权** | 上级可以管理所有下级账户 |
| **业务数据隔离** | 上级**不能**修改下级的业务配置 |
| **身份唯一性** | 同一用户在同一组织中只能有一个身份/角色 |
| **跨组织身份** | 同一用户在不同组织中可以有不同身份 |

### 3.3 数据权限范围

| 范围 | 说明 |
|------|------|
| **全部数据** | 可以看到所有数据（1级管理员） |
| **本组织及下级** | 可以看到本组织及其下级组织的数据 |
| **仅本人** | 只能看到自己创建/负责的数据 |
